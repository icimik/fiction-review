# 协作流程（PR 审稿制）

## 推荐主流程

1. 作者在新分支中提交稿件或修订内容
2. 发起 Pull Request，并说明本轮希望重点审阅的问题
3. 编辑基于 PR 进行文学编辑 / 出版编辑 / MBTI 读者评审
4. 将短版结论沉淀到 PR 评论
5. 将正式结论沉淀到 `<project>/reviews/` 文档
6. 后续修订继续通过新的 PR 迭代

## 适用原则

- 正式稿件默认走 PR 流程
- 聊天直发更适合片段、脑暴和临时问诊
- 默认作者提交原稿，编辑给出建议与判断
- 除非作者明确要求，否则不直接覆盖式代改正文

## PR 描述建议包含

- 本次提交内容类型
- 本轮希望重点审阅的问题
- 目标读者（若有）
- 是否需要 MBTI 评审
- 是否需要量化评分
- 是否需要出版视角判断
